Connections

ESP32 Development Board Overview

The ESP32 has several GPIO (General Purpose Input/Output) pins, which we'll use for connecting the sensors. The most important pins for each sensor are:

  • VCC (or 3.3V or 5V): Power supply pin.
  • GND: Ground pin.
  • Data pins: The signal pins from each sensor go to specific GPIO pins on the ESP32.

Here’s a guide to connect each sensor:

Connecting the DHT11 (Temperature and Humidity Sensor)

  • VCC: Connect to 3.3V pin on ESP32.
  • GND: Connect to GND on ESP32.
  • DATA: Connect the DATA pin to GPIO 4 (or any other GPIO pin you want to use).

Note: You will need to use a pull-up resistor (10kΩ) between the data line and the VCC to stabilize the signal.

Connecting the Infrared Obstacle Avoidance Sensor

  • VCC: Connect to 3.3V or 5V pin (depending on your IR sensor's voltage requirement) on ESP32.
  • GND: Connect to GND on ESP32.
  • OUT: Connect the OUT pin to GPIO 5 (or another GPIO pin).

Connecting the Sound Sensor (KY-038)

  • VCC: Connect to 3.3V or 5V on ESP32 (depends on the sensor).
  • GND: Connect to GND on ESP32.
  • DO (Digital Out): Connect the DO pin to GPIO 18.
  • AO (Analog Out): You can optionally connect the AO pin to an ADC-capable pin like GPIO 36 or 39 if you want analog data.

Connecting the MQ-2 Gas Sensor

  • VCC: Connect to 5V on ESP32 (MQ-2 requires 5V for accurate readings).
  • GND: Connect to GND on ESP32.
  • AO (Analog Output): Connect to GPIO 36 (which supports ADC).
  • DO (Digital Output): Connect to GPIO 19 (if you want digital values).

Connecting the ESP32-CAM (with ESP32-CAM-MB)

You can use the ESP32-CAM-MB to connect the ESP32-CAM module via USB. No extra wiring is needed between the ESP32-CAM and the development board. Simply plug in the USB connector.

Breadboard and Wiring

Using your MB102 breadboards, you can connect the VCC and GND of all components to the power and ground rails of the breadboard. Here's how:

  • Use one breadboard for power management. Connect the 3.3V and GND from the ESP32 to the power rails of this breadboard.
  • Distribute power to each sensor by connecting their VCC to the power rail and GND to the ground rail.
  • Use the second breadboard for organizing sensor connections to the ESP32 GPIO pins using jumper wires.

Connections Overview

  1. DHT11 Temperature and Humidity Sensor:
    • VCC (DHT11) → 3V3 (ESP32)
    • GND (DHT11) → GND (ESP32)
    • DATA (DHT11) → GPIO4 (ESP32)
  2. MQ-2 Gas Sensor:
    • VCC (MQ-2) → 5V (ESP32)
    • GND (MQ-2) → GND (ESP32)
    • AO (MQ-2) → GPIO36 (ESP32)
  3. Infrared Obstacle Avoidance IR Sensor:
    • VCC (IR Sensor) → 5V (ESP32)
    • GND (IR Sensor) → GND (ESP32)
    • OUT (IR Sensor) → GPIO14 (ESP32) (you can choose another GPIO pin as well)
  4. Sound Detection Module:
    • VCC (Sound Sensor) → 3V3 (ESP32)
    • GND (Sound Sensor) → GND (ESP32)
    • OUT (Sound Sensor) → GPIO13 (ESP32) (you can choose another GPIO pin as well)
  5. LED Connection:
    • Anode (+) of LED → GPIO12 (ESP32) (you can choose another GPIO pin as well)
    • Cathode (-) of LED → GND (ESP32)

Libraries and Program

Steps for Setting Up Arduino IDE:

Install ESP32 Board Package:

  • Open Arduino IDE.
  • Go to File → Preferences.
  • In Additional Board Manager URLs, add this link:
    • https://dl.espressif.com/dl/package_esp32_index.json
  • Then go to Tools → Board → Boards Manager, search for ESP32, and install it.

Install Necessary Libraries:

  • DHT Sensor Library: Go to Tools → Manage Libraries, search for DHT Sensor Library, and install it.
  • Adafruit Unified Sensor Library: This library will be automatically installed with the DHT library, but verify it’s installed.

Program Outline:

This program will:

  • Read temperature and humidity data from the DHT11.
  • Read values from the infrared obstacle sensor.
  • Detect sound from the sound detection module.
  • Read gas concentration (analog) from the MQ-2 sensor.
  • Display the sensor data to the Serial Monitor.

Code:

// Include libraries
#include   // DHT sensor library

// Define pin numbers
#define DHTPIN 4          // DHT11 data pin
#define DHTTYPE DHT11     // Define the type of DHT sensor
#define IR_SENSOR_PIN 16  // Infrared Obstacle Sensor
#define SOUND_SENSOR_PIN 17 // Sound Detection Module
#define GAS_SENSOR_PIN 36  // MQ-2 Gas Sensor (Analog)
#define LED_PIN 21        // LED

// Initialize DHT sensor
DHT dht(DHTPIN, DHTTYPE);

void setup() {
  // Initialize serial communication at 115200 baud rate
  Serial.begin(115200);

  // Initialize DHT sensor
  dht.begin();

  // Set pin modes
  pinMode(IR_SENSOR_PIN, INPUT);       // Infrared sensor as input
  pinMode(SOUND_SENSOR_PIN, INPUT);    // Sound sensor as input
  pinMode(GAS_SENSOR_PIN, INPUT);      // Gas sensor as input (analog)
  pinMode(LED_PIN, OUTPUT);            // LED as output

  // Initial LED state
  digitalWrite(LED_PIN, LOW);
}

void loop() {
  // Read temperature and humidity from DHT11
  float temperature = dht.readTemperature();  // Get temperature in Celsius
  float humidity = dht.readHumidity();        // Get humidity

  // Check if any reading from DHT failed
  if (isnan(temperature) || isnan(humidity)) {
    Serial.println("Failed to read from DHT sensor!");
  } else {
    // Print temperature and humidity data
    Serial.print("Temperature: ");
    Serial.print(temperature);
    Serial.println("°C");

    Serial.print("Humidity: ");
    Serial.print(humidity);
    Serial.println("%");
  }

  // Read from Infrared Obstacle Sensor (Active Low)
  int irSensorState = digitalRead(IR_SENSOR_PIN);
  if (irSensorState == LOW) {
    Serial.println("Obstacle detected!");
  } else {
    Serial.println("No obstacle.");
  }

  // Read from Sound Detection Module
  int soundSensorState = digitalRead(SOUND_SENSOR_PIN);
  if (soundSensorState == HIGH) {
    Serial.println("Sound detected!");
  } else {
    Serial.println("No sound detected.");
  }

  // Read from MQ-2 Gas Sensor (Analog)
  int gasSensorValue = analogRead(GAS_SENSOR_PIN);
  Serial.print("Gas sensor value: ");
  Serial.println(gasSensorValue);

  // Control the LED (turn it on if gas value exceeds a threshold)
  if (gasSensorValue > 400) {  // Set a threshold value for gas detection
    digitalWrite(LED_PIN, HIGH);
    Serial.println("LED ON - High gas concentration detected!");
  } else {
    digitalWrite(LED_PIN, LOW);
    Serial.println("LED OFF - Gas concentration normal.");
  }

  // Delay before the next loop iteration
  delay(2000);  // 2 seconds delay between readings
}

Code Explanation:

Library Inclusions:

The DHT.h library handles the DHT11 sensor data. Make sure you install the DHT library before compiling.

Pin Definitions:

We define the pin numbers for the DHT11, IR sensor, sound module, MQ-2 gas sensor, and an LED to show a visual signal when gas concentration exceeds a threshold.

Setup:

Initializes the DHT sensor and sets the pin modes for each sensor and the LED. The serial communication is started to display sensor data on the Serial Monitor.

Main Loop:

Reads data from each sensor:

  • DHT11 for temperature and humidity.
  • Infrared sensor for obstacle detection.
  • Sound detection module for any noise.
  • MQ-2 gas sensor for gas concentration.

The data is printed to the Serial Monitor for each sensor. The LED is turned ON if gas levels exceed a preset threshold (you can adjust this value according to your needs).

How to Run the Program:

  • Connect ESP32 to your PC using a USB cable.
  • Open the Arduino IDE and copy the provided code into a new sketch.
  • Go to Tools → Board → ESP32 Dev Module and select the correct port (COM port for Windows or /dev/ttyUSB0 for Linux).
  • Click on the Upload button to compile and upload the code to your ESP32.
  • Open the Serial Monitor by going to Tools → Serial Monitor and set the baud rate to 115200.
  • Once the code is running, you should see sensor data output in the Serial Monitor.
